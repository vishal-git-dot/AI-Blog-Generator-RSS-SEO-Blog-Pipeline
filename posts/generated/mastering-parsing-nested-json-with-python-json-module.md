When the json module reads a string, it builds a tree of Python dict and list objects that mirrors the original hierarchy. Each dict key becomes a str , each numeric literal becomes int or float , and so on. Under the hood the parser scans the character stream, identifies structural tokens (braces, brackets, commas, colons), and allocates Python objects on the heap. Because each level creates a new object, deeply nested structures can generate a cascade of allocations, which is why a clear traversal strategy matters for both performance and error handling.
