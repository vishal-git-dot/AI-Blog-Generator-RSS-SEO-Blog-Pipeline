# How to Write a Script That Finds Every Parameterized URL Variant on Your Site

## Overview

Before you can fix duplicate content caused by URL parameters, you need to know the actual scope. Guessing from a few examples in Search Console misses most of the problem. This walks through a small Python script that parses server logs and groups requests by their base path, so you can see exactly how many parameter variants each route is generating. Why log parsing beats Search Console alone Search Console tells you which duplicate issues Google noticed, but it samples and aggregates, so smaller-volume parameter combinations don't always surface individually. Your own server logs (or a CDN's access logs) contain every request that actually happened, which gives you the real count. Extending it to check canonical tags directly Once you have a list of the worst offending base paths, cross-reference a sample of each parameter variant's actual canonical tag using a lightweight request: import requests from bs4 import BeautifulSoup def get_canonical ( url ): resp = requests . get ( url , timeout = 10 ) soup = BeautifulSoup ( resp . text , " html.parser " ) tag = soup . find ( " link " , rel = " canonical " ) return tag [ " href " ] if tag else None Run this against ten or so parameter variants per base path and confirm they all resolve to the same canonical URL. Filtering out bot traffic that isn't Googlebot Not every request in your logs represents crawl budget you care about optimizing.
