Challenges and Solutions Challenge 1: Coordinate Mapping MediaPipe returns landmark positions as normalized values between 0 and 1. Rendering them on a canvas requires mapping these to actual pixel coordinates. Solution: Multiply the normalized x and y values by canvas.width and canvas.height respectively. const x = landmark . x * canvas . width ; const y = landmark . y * canvas . height ; Challenge 2: Canvas Mirroring When using the front-facing webcam, the video appears mirrored — but the canvas wasn't. This created a disconnect between where your hands physically are and where the effect renders. Solution: Apply a CSS transform to flip the video and canvas simultaneously: video , canvas { transform : scaleX ( -1 ); } Challenge 3: Model Loading Latency MediaPipe's WASM model takes a moment to load on the first visit, which can feel like the app is broken. Solution: Show a loading indicator until the model is ready and the first frame is processed.
