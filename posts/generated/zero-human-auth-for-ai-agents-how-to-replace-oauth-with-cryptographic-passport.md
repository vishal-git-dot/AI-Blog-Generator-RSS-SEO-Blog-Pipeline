# Zero Human Auth for AI Agents: How to Replace OAuth with Cryptographic Passport

## Overview

Your AI agent needs to log into a customer's dashboard at 3 a.m. You cannot ask a human to click Allow . The agent has no browser. It cannot complete an OAuth redirect. So you pick one of these: Store an immortal API key in the worker (and hope it never leaks) Run headless Chrome with Playwright (and fix it every time the login UI changes) Use Device Authorization Grant (and wait for someone to scan a QR code) These are duct tape. They do not give you AI agent identity — a signed, auditable proof of which agent logged in, bound to a specific site session, with a separate lane for MCP OAuth on external tool servers. Zero Human Auth means the human is not in the login path. It does not mean "no authentication."
